DbUtils使用方法

使用DbUtils可以省去创建数据库建表，写数据库操作工具类等等各种重复没意思的东西，只需要以下步骤，就可以完成OOM（对象关系映射），用代码直接建数据库建表了。

以学生Student实体为例，用DbUtils进行增删改查操作。

1.创建Student.class
    要包含一个字段_id(long)，用注解修饰（@Id），标识主键
    其余字段（date，money，category...）用注解@Colum修饰（映射为列）
    get，set方法照旧
2.在代码合适的地方创建数据库
    /**
    *参数1：上下文
    *参数2：数据库名称
    **/
    DbUtils dbUtils=DbUtils.create(context,"app.db")
3.基本简单操作

    3.1：增
        Student s=new Student();
        .....
        给Student对象赋值
        .....
        //调用save（）方法保存单个对象
        dbUtils.save(s);
    3.2:删
        (1):
            简单删除
            dbUtils.delete(s)//s中必须包含id并且有值
        （2） 带条件的删除
            /**
            *参数1：删除数据所在表
            *参数2：条件
            **/
            WhereBuilder.b("列","操作"，"值")生成WhereBuilder
                        .and("列","操作"，"值")and
                        .or("列","操作"，"值")or
                        .expr("列","操作"，"值")特殊表达式
                        .expr("完整的条件表达式");
            dbUtils.delete(Class表，WhereBuilder);
     3.3:改  

     简单修改：
    dbUtils.saveOrUpdate(Object entity)//如果实体中包含@id的内容并且              有值，则可以以此为条件进行调用，因此能够更新单条记录。如果不包含（没有主键），就是添加
    带条件的修改：
    dbUtils.update(Object entity,WhereBuilder,String...要更新的字段列表)        
     3.4：查
        dbUtils.findAll(Class entityType)//获取或有的数据，返回特定类型的集合
        Selector.from(Class 表).where(列，操作，值)//"price","=","100"
                .and(列，操作，值)
                .or(列，操作，值)
                .groupBy(String)
                .orderBy(String)

ViewUtils的简单使用

注解的描述信息：
            ViewUtils.inject(Activity)动态加载，分析注解
@ContentView(R.layout.xxx)：描述setContentView
@ViewInject（R.id.xxx）:描述findViewById
@OnClick（R.id.xxx）:给xxx描述事件监听
@ResInject(id=R.xx.x,type=ResType.xxx):描述各种资源

视图注解注册的时机：

Activity  onCreate 中 调⽤用 ViewUtils.inject(Activity)
Fragment onCreateView中，调⽤用 ViewUtils.inject(fragment, View)
inject ⽅方法，名称叫做“注射”，将实际的控件或者是内 容，通过反射的形式，注射到   成员变量中。 ViewUtils inject() ⽅方法，实际上，内部采⽤用反射的形式， 将类中的所有的注解全部获取到，然后，根据 ViewInject 那么执⾏行ﬁndViewById，根据 ResInject 进行资源的获取。

HttpUtils的简单使用 
1.每次进行连接的时候，需要创建一个HttpUtils的对象 
2.如果进行多线程的请求，那么必须一个请求一个HttpUtils。

内部包含异步任务的实现
支持同步，异步网络的请求
同步：需要在线程中请求
异步：使用自身的线程进行网络请求，需要接口回调。

Get请求
send 是在异步任务中进⾏行请求，需要更新UI的，可以 ⽤用这个⽅方法，就像普通的异步任务； sendSync() 需要在⼦子线程中进⾏行执⾏行；可以在不需要 更新的情况下，例如后台下载

BitMapUtils的简单使用

 // TODO 图片加载

    // 1. 创建 BitmapUtils 工具实例，可以认为就是一个 ImageLoader
    //    构造方法有很多，包含 文件缓存配置、内存缓存配置，避免内存溢出、提高图片加载的速度
    //    减少网络流量
    bitmapUtils = new BitmapUtils(this);

    // 2. display 用于 加载、显示图片

    BitmapDisplayConfig displayConfig = new BitmapDisplayConfig();

    // 2.1 设置加载中图片
    Resources resources = getResources();

    displayConfig.setLoadingDrawable(resources.getDrawable(R.drawable.ic_launcher));

    // 2.2 设置加载失败的图片
    displayConfig.setLoadFailedDrawable(resources.getDrawable(android.R.drawable.ic_delete));

    // 2.3 设置 Bitmap 像素的配置
    displayConfig.setBitmapConfig(Bitmap.Config.RGB_565);

    // 2.4 设置Bitmap最大的尺寸
    BitmapSize size = new BitmapSize(100, 50);
    displayConfig.setBitmapMaxSize(size);

    // 2.4 设置，图片加载成功之后，到最终图片显示完成之间的动画
    displayConfig.setAnimation(animation);

    // 第一个参数: 范型 如果是 ImageView ，就显示图片，否则其余 View ，那么显示背景
    bitmapUtils.display(
            imageView,
            "http://bbs.lidroid.com/static/image/common/logo.png",
            displayConfig
    );
